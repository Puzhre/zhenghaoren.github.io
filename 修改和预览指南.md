# 个人网站修改和预览指南

## 📝 如何修改个人信息

### 1. 修改 `_config.yml` 文件

这是最重要的配置文件，包含网站的基本信息和您的个人信息。

**需要修改的关键字段：**

- **第 12 行** `title`: 网站标题（例如："张三的个人网站"）
- **第 14 行** `name`: 您的姓名
- **第 15 行** `description`: 网站描述
- **第 16 行** `url`: 改为您的 GitHub Pages 地址（例如：`https://zhenghaoren.github.io`）
- **第 19 行** `repository`: 改为您的仓库名（例如：`zhenghaoren/zhenghaoren.github.io`）

**作者信息部分（第 24-83 行）：**

```yaml
author:
  # 基本信息
  avatar           : "profile.png"  # 头像文件名（放在 images/ 目录）
  name             : "您的姓名"
  pronouns         : # 可选，例如："she/her"
  bio              : "左侧边栏的简短个人简介"
  location         : "您所在的城市"
  employer         : "您的单位/学校"
  email            : "your.email@example.com"
  
  # 学术网站链接（填写您的实际链接）
  googlescholar    : "https://scholar.google.com/citations?user=YOUR_ID"
  orcid            : "https://orcid.org/YOUR_ORCID"
  github           : "您的GitHub用户名"
  
  # 社交媒体（根据需要填写）
  linkedin         : "您的LinkedIn用户名"
  twitter          : "您的Twitter用户名"
  zhihu            : "您的知乎用户名"
  weibo            : "您的微博用户名"
```

### 2. 修改 `_data/authors.yml` 文件

这个文件包含作者的详细信息，用于文章和页面的作者信息显示。

### 3. 修改 `_pages/about.md` 文件

这是"关于"页面的内容，您可以在这里写详细的自我介绍。

### 4. 替换头像图片

将您的头像图片放到 `images/` 目录下，命名为 `profile.png`（或修改 `_config.yml` 中的 `avatar` 字段指向您的图片文件名）。

## 🚀 如何预览网站

### 方法 1：使用 Jekyll 本地服务器（推荐）

**前提条件：**
- 安装 Ruby 和 Bundler
- Windows 用户可能需要安装 Ruby DevKit

**步骤：**

1. **安装依赖**
   ```bash
   bundle install
   ```
   如果遇到权限错误，可以设置本地安装路径：
   ```bash
   bundle config set --local path 'vendor/bundle'
   bundle install
   ```

2. **启动本地服务器**
   ```bash
   bundle exec jekyll serve -l -H localhost
   ```
   或者：
   ```bash
   jekyll serve -l -H localhost
   ```

3. **访问网站**
   打开浏览器访问：`http://localhost:4000`

   - 修改 Markdown 文件（`.md`）后，页面会自动刷新
   - 修改 `_config.yml` 后，需要重启服务器（按 `Ctrl+C` 停止，然后重新运行命令）

### 方法 2：使用 Docker（适合不想安装 Ruby 的用户）

**前提条件：**
- 安装 Docker Desktop

**步骤：**

1. **启动 Docker 容器**
   ```bash
   docker compose up
   ```

2. **访问网站**
   打开浏览器访问：`http://localhost:4000`

### 方法 3：使用 VS Code DevContainer

如果您使用 VS Code：
1. 打开命令面板（F1）
2. 选择 "DevContainer: Reopen in Container"
3. 网站会自动在 `http://localhost:4000` 上运行

## 📋 修改清单

在修改个人信息时，建议按以下顺序进行：

- [ ] 修改 `_config.yml` 中的基本信息和作者信息
- [ ] 修改 `_data/authors.yml` 中的作者详情
- [ ] 修改 `_pages/about.md` 中的关于页面内容
- [ ] 替换 `images/profile.png` 为您的头像
- [ ] 更新 `_config.yml` 中的 URL 和仓库名
- [ ] 本地预览检查所有修改
- [ ] 提交并推送到 GitHub

---

## ⚠️ 注意事项

1. **修改 `_config.yml` 后必须重启 Jekyll 服务器**才能看到效果
2. **YAML 格式敏感**：注意缩进和冒号后的空格
3. **图片路径**：所有图片应放在 `images/` 目录下
4. **URL 格式**：确保 URL 以 `https://` 开头，末尾不要有斜杠

## 🔍 常见问题

**Q: 修改后看不到变化？**
A: 如果修改的是 `_config.yml`，需要重启服务器。如果修改的是 Markdown 文件，等待几秒让 Jekyll 自动重新生成。

**Q: 端口 4000 被占用？**
A: 可以使用其他端口：`jekyll serve -l -H localhost -P 4001`

**Q: 如何查看错误信息？**
A: 查看终端输出的错误信息，通常会有详细的错误提示。
